# BUG001: CallGraph 동명이패키지 함수 충돌

## 증상

```bash
filefunc chain func ParseFile --chon 1 --meta what
```

기대 결과 (funcspec):
```
ParseFile (what="단일 func spec .go 파일을 파싱하여 FuncSpec을 반환한다")
  1촌 calls: parseCommentGroup
  1촌 calls: processDecl
  1촌 called-by: ParseDir
```

실제 결과 (stml/parser):
```
ParseFile (what="단일 HTML 파일을 파싱하여 PageSpec 반환")
  1촌 calls: ParseReader
  1촌 called-by: ParseDir
  1촌 called-by: parseDirEntry
```

`--package`로 한정해도 실패:
```bash
filefunc chain func ParseFile --package funcspec --chon 1 --meta what
# Error: func "ParseFile" not found in package "funcspec" (found in "parser")
```

## 영향 범위

fullend 프로젝트 기준 동명 함수:

| 함수명 | 존재 패키지 (5개) |
|---|---|
| ParseFile | funcspec, stml/parser, ssac/parser, policy, statemachine |
| ParseDir | funcspec, stml/parser, ssac/parser, statemachine, policy, scenario |

chain 결과의 호출 관계, `//ff:what` 메타데이터가 모두 잘못된 패키지 것으로 표시된다.

## 근본 원인

`CallGraph`와 `BuildFuncFileMap`이 함수명만 키로 사용:

```go
// call_graph.go
type CallGraph struct {
    Children map[string][]string  // "ParseFile" → callees
    Parents  map[string][]string  // "ParseFile" → callers
}

// func_file_map.go
func BuildFuncFileMap(files []*model.GoFile) map[string]*model.GoFile {
    m := make(map[string]*model.GoFile)
    for _, gf := range files {
        for _, name := range gf.Funcs {
            m[name] = gf  // 동명 함수 → 마지막 등록이 덮어씀
        }
    }
    return m
}
```

WalkDir 순서상 마지막에 등록된 패키지가 map을 덮어쓴다. `--package` 필터는 구현되어 있지만, 필터링할 대상 자체가 이미 덮어써진 상태라 동작하지 않는다.

## 수정 방향

CallGraph 키를 `pkg.FuncName` 형식으로 변경:

```go
type CallGraph struct {
    Children map[string][]string  // "funcspec.ParseFile" → callees
    Parents  map[string][]string  // "funcspec.ParseFile" → callers
}
```

영향받는 파일:
- `internal/chain/call_graph.go` — 키 형식 변경
- `internal/chain/build_call_graph.go` — 키 생성 시 패키지명 포함
- `internal/chain/func_file_map.go` — 동일
- `internal/chain/func_name.go` — 패키지명 반환 추가
- `internal/chain/traverse_chon.go` — 키 형식 대응
- `internal/chain/traverse_depth.go` — 동일
- `internal/chain/format_chain.go` — 출력 시 패키지명 표시/생략
- `internal/cli/chain_func.go` — `--package` 미지정 시 동명 함수 목록 안내

## 재현

```bash
cd ~/.clari/repos/fullend
filefunc chain func ParseFile --chon 1 --meta what
# → stml/parser의 ParseFile이 출력됨

filefunc chain func ParseFile --package funcspec --chon 1 --meta what
# → Error: func "ParseFile" not found in package "funcspec" (found in "parser")
```
